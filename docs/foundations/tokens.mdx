---
title: Tokens
description: Design tokens are all the values needed to construct and maintain a design system —spacing, color, typography, object styles, animation, etc. — represented as data.These can represent anything defined by design color as an RGB value, an opacity as a number, animation ease as Bezier coordinates. They're used in place of hard-coded values to ensure flexibility and unity across all product experiences. In addition, this allows for better-shared control between designers and engineers.

## Size

Size tokens are relative to the [baseSize](/foundations/tokens#Base) and are used when a specific fixed width or height is desired when designing a component or layout.

```jsx
// preview
import { styled, theme } from "@washingtonpost/wpds-ui-kit";

export default function Example() {
  const SizeExample = styled("div", {
    width: theme.sizes["300"], // Size example
    height: theme.sizes["300"], // Size example
    backgroundColor: theme.colors.primary,
  });
  return <SizeExample />;
}
```

## Space

Space tokens are relative to the [baseSize](/foundations/tokens#Base) and are used to define padding or margin when designing a component or layout.

## Fonts

Font tokens define an intention and pairing to a font-family.

```jsx
// preview
import { styled, theme } from "@washingtonpost/wpds-ui-kit";

export default function Example() {
  const Headline = styled("h1", {
    fontFamily: theme.fonts.headline, // font family example
    fontSize: theme.fontSizes.["200"],
    color:theme.colors.primary,
  });
  const Subhead = styled("h2", {
    fontFamily: theme.fonts.subhead, // font family example
    fontWeight:theme.fontWeights.light,
    fontSize: theme.fontSizes.["150"],
    color:theme.colors.accessible,
  });
  const Body = styled("p", {
    fontFamily: theme.fonts.body, // font family example
    fontSize: theme.fontSizes.["100"],
  });
  const Meta = styled("p", {
    fontFamily: theme.fonts.meta, // font family example
    fontSize: theme.fontSizes.["075"],
    color:theme.colors.accessible,
  });
  return (
    <div>
      <Headline>Headline</Headline>
      <Subhead>Subheadline</Subhead>
      <Body>
        Body text
      </Body>
      <Meta>
        Meta text
      </Meta>
    </div>
  );
}
```

## Line heights

Line height tokens are unitless values that define the height of a text element based on its current font size.

```jsx
// preview
import { styled, theme } from "@washingtonpost/wpds-ui-kit";

export default function Example() {
  const LineHeightExample = styled("p", {
    fontFamily: theme.fonts.body,
    lineHeight:theme.lineHeights["125"],
    fontSize: theme.fontSizes.["125"],
    color:theme.colors.primary,
    width:250
  });

  return (
      <LineHeightExample>Lorem ipsum dolor sit amet, consectetur
        adipiscing elit.
```

## Color

All color tokens define the color in the current context. There are 2 contexts `light` and `dark` context.
Toggle the sites context to see the color values for each context in the example.

```jsx
// preview
import { styled, theme } from "@washingtonpost/wpds-ui-kit";

export default function Example() {
  const ColorExample = styled("p", {
    backgroundColor: theme.colors.gray20, //Example of color token
    color: theme.colors.gray600, //Example of color token
    padding: theme.space["100"],
  });

  return (
    <ColorExample>
      Toggle to see different color values for each context

  );
}
```

### Static color tokens

Static color tokens remain static regardless of the current context. To use a static token simple apply a `-static` to the color token. All static colors are based in the `light` context.

```jsx
// preview
import { styled, theme } from "@washingtonpost/wpds-ui-kit";

export default function Example() {
  const ColorExample = styled("p", {
    backgroundColor: theme.colors["gray20-static"], //Example of static color token
    color: theme.colors["gray600-static"], //Example of static color token
    padding: theme.space["100"],
  });

  return <ColorExample>Toggle to see how static colors work</ColorExample>;
}
```

### Theme and Semantic tokens

Theme tokens are alias tokens that are mapped to a color token. They help define intention around
how we use color. Majority of our designs should use our theme tokens as much as possible. Read more
about how to use theme and semantic tokens on our [color documentation](/foundations/color#Themes).

```jsx
// preview
import { styled, theme } from "@washingtonpost/wpds-ui-kit";

export default function Example() {
  const ColorExample = styled("p", {
    backgroundColor: theme.colors.primary, //Example of static color token
    color: theme.colors.onPrimary, //Example of static color token
    padding: theme.space["100"],
  });

  return <ColorExample>Theme token</ColorExample>;
}
```

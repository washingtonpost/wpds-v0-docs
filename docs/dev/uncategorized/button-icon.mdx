# Buttons

<!-- Description of component -->

Buttons allow users to perform an action with a single tap. 
Buttons also serve as way for a call to action.

## Accessibility

### Writing an ARIA label

Because the `ButtonIcon` has no visible text, an ARIA label is that much more important to help users who user screen readers to understand the action of the button. Be sure to pass a sting via the `ariaLabel` prop that describes the resulting action of the button.

**Examples of button aria labels:**

- "Add to your reading list"
- "Copy story link to clipboard"
- "Close"
- "Play audio story"

**Learn more:**

- [Creating Accessible Icon Buttons](https://www.sarasoueidan.com/blog/accessible-icon-buttons/)
- [W3G: Using aria-label to provide an invisible label where a visible label cannot be used](https://www.w3.org/TR/WCAG20-TECHS/ARIA14.html)
- [Make Your ARIA Labels Sing on Key](https://knowbility.org/blog/2019/ARIA-labels/)

### Adding role="button" when rendering as a link

It is possible to render the `ButtonIcon` as a link or an `a` element, as some buttons in designs are actually links. If you chose to pass the `as` prop with `"a"` then you should also add the `role="button"` to the `ButtonIcon`.
